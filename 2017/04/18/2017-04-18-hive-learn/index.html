hive体系架构

hive组成模块有

  • CLI（命令行界面）
  • HWI（Hive网页界面）
  • Thrift (连接Hive的驱动服务器)
  • Driver (驱动模块，Hql解释器，优化器,MR执行计划在这个模块)
  • Metastore (元数据存储,存放表结构和字段信息，一般是关系型数据库)
  • HDFS (分布式文件系统，存放数据使用)

其中最常用的就是CLI界面，HWI界面几乎不用
如果想要通过JDBC,ODBC等方式连接Hive进行编程访问，需要开启hiveserver服务

hiveserver2是hiveserver的升级版，但是hiveserver2不兼容hiveserver，所以需要同时开启两个服务(记住开启不同端口),或者只使用hiveserver

然后hql的解释器，执行器都在Driver模块中，hive通过和jobTracker通信来初始化MR任务(job),Hive是通过一个”job执行计划”的XML文件驱动驱动执行内置的MR模块。

Metastore和HDFS都是依赖外部程序，类似Mysql，Hadoop等，简单的说，hive就是把好写的hive的sql转换为复杂难写的map-reduce程序的一个程序。

hive 基本数据类型

基本数据类型

数据类型 长度 范围 说明
tinyint 1byte -128 ~ 127 有符号整数
smalint 2byte -32,768 ~ 32,767 有符号整数
int 4byte -2,147,483,646 ~ 2,147,483,647 有符号整数
BIGINT 8byte 有符号整数
BOOLEAN 布尔类型 TRUE/FALSE TRUE/FALSE
FLOAT 4byte 单精度浮点型
DOUBLE 8byte 双精度浮点型
STRING 2G个字符 使用int做字符索引
BINARY 字节数组 从Hive0.8.0开始支持
TIMESTAMP 整数,浮点数或字符串 ‘2012-02-03’ 或 1327882394(unix新纪元秒)
DECIMAL 从Hive0.11.0开始支持
CHAR CHAR类型与VARCHAR类型相似，但拥有固定的长度，也就是如果字符串长度小于指示器的长度则使用空格填充。CHAR类型的最大长度为255 从Hive0.13.0开始支持
VARCHAR 可变长字符串(1~65355) 从Hive0.12.0开始支持
DATE 从Hive0.12.0开始支持

需要注意的是所有的这些数据类型都是对Java中的接口的实现，因此这些类型的细节和Java中的对应类型是完全一致的。

复杂数据类型

数据类型 描述 实例
STRUCT STRUCT可以包含不同数据类型的元素。这些元素可以通过”点语法”的方式来得到所需要的元素，比如user是一个STRUCT类型，那么可以通过user.address得到这个用户的地址。 STRUCT(col_name : data_type [COMMENT col_comment], …)
MAP MAP包含key->value键值对，可以通过key来访问元素。比如”userlist”是一个map类型，其中username是key，password是value；那么我们可以通过userlist[‘username’]来得到这个用户对应的password； MAP(primitive_type, data_type)
ARRAY ARRAY类型是由一系列相同数据类型的元素组成，这些元素可以通过下标来访问。比如有一个ARRAY类型的变量fruits，它是由[‘apple’,’orange’]组成，那么我们可以通过fruits[1]来访问元素orange，因为ARRAY类型的下标是从0开始的； ARRAY(data_type)
UNION UNIONTYPE，他是从Hive 0.7.0开始支持的。 UNIONTYPE(data_type, data_type, …)
