利用github + hexo 搭建个人博客

发表于 2017-03-09 | | 阅读次数

　　写博客的初衷已经在博客初识中说过一遍，在此便不再赘述。
　　这篇博客记录的是我自己从无到有搭建属于自己的个人网站的过程，提供给各位做一些参考

环境:window 7 x64
git
node.js
markdownPad

  • 注册github账号，创建博客版本库

注册github创建博客版本库

注册github的过程不过多叙述。登陆github

  1. 创建一个版本仓库 点击 ”New repository”

  1. 版本库名称为 username.github.io username是github的账号名，是github官方为每一个用户预留的域名地址，可以直接通过username.github.io 访问自己的博客。

  1. 默认设置，创建版本库

注：后续步骤可选

*4. 进入username.github.io版本库 选择setting

*5. 找到Github pages 模块 这里选择 change theme

选择一个theme 然后submit

*6. 在浏览器中打开 username.github.io 即可看到自己的博客

注册并绑定域名

参考博客 GitHub Pages 绑定来自阿里云的域名

github可以给自己的博客绑定一个域名，我们购买一个域名即可

购买域名在阿里云上购买即可

控制台->域名与网站->域名->域名注册

注册了之后需要实名验证才可以使用域名

然后在 blog/source/目录下 添加一个文件CNAME
内容为你的域名
your domain

然后重新发布 CNAME文件就会出现在github的根目录下

绑定完成后，登录阿里云单域名控制台，在域名解析中添加如图所示的解析

然后在浏览器输入该域名访问博客

注意1.第一个A和第二个A ip地址不同
第一个A对应的ip是 192.30.252.154
第二个A对应的ip是 192.30.252.153
最上面的CNAME 对应的域名 username.github.io. ，最后的.不能忘记

异常处理

发布成功，博客内本地图片无法访问异常

hexo 中 image,js,这种资源文件全部放在blog/source/文件夹下，然后在本地使用markdownpad编辑md文件时，使用图片的路径为

![](../images/img.jpg)

才能预览图片
但是如果要发布到网页上，需要将路径改动成根目录路径
