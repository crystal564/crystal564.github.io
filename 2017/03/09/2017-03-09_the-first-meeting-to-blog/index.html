初识

发表于 2017-03-09 | | 阅读次数

初识github + hexo搭建的个人博客的魅力, 于是在搭建之初记录下了自己的心情和对博客的一个期望。

　　大二就听说了github,当时还不了解github能够用来做什么，为了跟风跑去注册了一个账号，之后便再也没有登陆过。
　　直到2016年的8月份，刚开始实习，接触了很多开源软件，然后就想起了当初的github账号，试了很久的密码才登陆进去，创建了第一个Repository，不过后来又匿了。
　　等到了2017年的3月份，实习的公司资金链断裂，面对寻找工作的压力的时候，回想自己过去几个月的收获，突然有种冲动，觉得自己该写点什么，记录下来某些东西。于是就动起了写Blog的想法。
　　大二学习算法的时候，因为在网上看了很多大神的CSDN的博客，学习了很多，也发过几篇CSDN博客。所以写博客的第一反应就是去CSDN上写。但是脑海里突然想起了之前在知乎上看到过的有人用github搭建的免费的个人博客，觉得非常酷，于是又去网上查资料研究。
　　研究了以后发现github上虽然空间免费，还可以绑定域名，但是只能用静态网页，而且用户对象也都是一些专业的大神，觉得没有太多的人会看到自己的博客，就在CSDN和github犹豫抉择，直到在知乎的一个帖子上看到了大神的博客Hux Blog | huxpro.github.io。我才惊讶的发现，静态网页居然也可以做的和动态网页一般。不用自己操心写html代码的事，而且效果非常不错，于是我就决定在github上写Blog。

需求分析

　　既然要写Blog，那么便做好准备功课，首先，一个人无论做什么事情都会有一个动机，那么我写Blog的动机是什么，是想要获得人气？还是想要获得心理慰藉？还是…？

美国心理学家亚伯拉罕·马斯洛在1943年在《人类激励理论》论文中所提出。书中将人类需求像阶梯一样从低到高按层次分为五种，分别是：生理需求、安全需求、社交需求、尊重需求和自我实现需求。– 摘自 百度百科<<需求层次理论>>

　　最后，我得出的结论是，我的需求就是:自我实现，我的自我实现不需要在意外人的眼光，我做了我觉得喜欢的事情，便是自我实现，所以我的Blog也应当如此，用于记录我的生活，记录我一路的历程和学习。这就是我写博客的初衷。

整体设计

　　在决定了使用github搭建博客，并且博客的目的是为了记录我生活的点滴之后，就开始着手设计这个博客了。它是什么样式的，有着什么样的主题，不想花太多时间在设计html，css，js上还需要好看需要如何去设计。于是我又开始了新一轮的查资料。之前提到的大神的博客Hux Blog，研究之后发现是利用github pages + Jekyll + markdown + git 实现的博客，jekyll是一款开源的简单静态博客网站生成器，类似这样的软件还有hexo，octopress等等，可以直接套用模板，然后将精力全部放在写博客上，无需在html，css上耗费大量时间。
　　在了解了之后，我有陷入了纠结，是选用jekyll还是hexo还是octopress呢，静态博客框架之Hexo & Jekyll 中对比了jekyll和hexo的优缺点和适用环境，最后我选择github + hexo + git + markdown搭建我的个人博客。

总结

　　具体的搭建步骤我会写成博客，以此纪念。在总结里不知道该写什么，突然想到了这篇文章里 静态博客框架之Hexo & Jekyll 阮一峰老师说的一段话

第一阶段，刚接触Blog，觉得很新鲜，试着选择一个免费空间来写。
第二阶段，发现免费空间限制太多，就自己购买域名和空间，搭建独立博客。
第三阶段，觉得独立博客的管理太麻烦，最好在保留控制权的前提下，让别人来管，自己只负责写文章。

　　虽然我只是刚开始接触Blog，但是并不妨碍我对第三阶段的幻想，所以在见到github+hexo搭建的静态博客系统之后，我便毫不犹豫的决定投身于github的怀抱，并在之后的时间里好好体会github的魅力。

黄俊辉 于17年03月 北京
